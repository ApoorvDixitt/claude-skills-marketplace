# Virality Frameworks — X Writing Reference

Load this file when composing threads, optimizing hooks, or planning content strategy.

---

## Algorithm weights (open-sourced, 2025)

Real scoring from X's open-source algorithm code. Not all engagement is equal.

| Signal | Weight vs. a like |
|---|---|
| Reply chain where author replies back | 150× |
| Retweet / repost | 20× |
| Profile click | 12× |
| Reply (no author reply-back) | 13.5× |
| Bookmark | 10× |
| Like | 1× |

**Key implications:**
- Replying to every comment in the first 30 min is your highest-leverage action after posting.
- Write for bookmarks (save-worthy info) and replies (opinion, question, controversy), not likes.
- External links on non-Premium = zero median engagement. Put links in a reply to your own thread.
- 3+ hashtags = 40% algorithmic penalty. Use 1–2 niche-specific tags or none.
- "Engagement bait" ("like if you agree") = explicit penalty. Use genuine questions instead.
- Follower count barely matters. Engagement rate drives impressions.
- First 30–60 minutes determines whether the algo expands reach to non-followers.

## Thread structure

The anatomy of a high-performing thread:

```
Tweet 1: HOOK
   └── Bold claim / curiosity gap / painful truth
   └── Works standalone. No context needed.
   └── Under 220 characters. Add "🧵" if it's a thread.

Tweet 2: CONTEXT ANCHOR
   └── Ground the reader. Who are you, why does this matter.
   └── Short. This earns trust, not attention.

Tweets 3–6: BODY
   └── One idea per tweet.
   └── Each tweet creates implicit tension → pulls to next.
   └── Never close a loop until the payoff.
   └── Vary sentence length. Short punch. Longer one to earn it.
   └── Visual break every 3–4 tweets (image, screenshot, chart).

Tweet 7: PAYOFF
   └── The single most quotable, saveable line.
   └── Short. Memorable. Could stand alone as a tweet.
   └── Write this FIRST — then build the thread backward from it.

Tweet 8: CTA
   └── Repost ask + follow ask + reply-driving question.
   └── "Which of these surprised you most?" beats "What do you think?"
   └── Put any links here, not in the main thread.
```

**Sweet spot:** 7 tweets (excluding CTA). Beyond 10 = completion drops sharply.

## The 6 frameworks

### Framework 1 — Hook → Story → Lesson → CTA
**Best for:** Personal experience, founder stories, career lessons, "I tried this" content.

```
Hook:   "[Shocking result or painful admission] 🧵"
Story:  What actually happened, step by step. Specific details only.
Lesson: The distilled insight — short, memorable, screenshot-worthy.
CTA:    "What would you have done differently?"
```

Example hook: "I posted every day for 60 days and lost 200 followers. Here's what changed when I stopped."

---

### Framework 2 — Contrarian take + proof
**Best for:** Thought leadership, standing out in a crowded niche, hot takes with receipts.

```
Hook:   "Everyone says [common advice]. They're wrong."
Tweet 2: "Here's why [mainstream belief] is backwards:"
Body:   Evidence, case studies, your own data, reasoning.
Payoff: The reframe in one sentence.
CTA:    "Agree? Disagree? Tell me why."
```

Example hook: "The best Twitter advice is 'post consistently.' It's also the most useless advice."

---

### Framework 3 — Listicle (numbered value drops)
**Best for:** Tools, tactics, mental models, "X things I wish I knew." High bookmark rate.

```
Hook:   "[N] [things] that [specific outcome]. I use all of them:"
Body:   One item per tweet. Each self-contained and useful.
CTA:    "Which one are you adding first?"
```

Example hook: "8 tools that cut my research time by 70%. All free. All underrated:"

---

### Framework 4 — Before → After → Bridge (BAB)
**Best for:** Transformation content, startup journey, skill acquisition, growth stories.

```
Hook:   "[Timeframe] ago: [painful before state].
        Today: [impressive after state].
        The only thing that changed 🧵"
Bridge: The exact steps between before and after. Specific, not vague.
```

Example hook: "90 days ago: 340 followers, 0 DMs. Today: 11K followers, 3 inbound deals this week. The only thing that changed:"

---

### Framework 5 — Problem → Agitate → Solution (PAS)
**Best for:** Educational content, explainers, skill-building threads.

```
Problem:  Name the specific pain your audience has.
Agitate:  Make them feel it. Show the cost of not solving it.
Solution: Walk through the fix with specific steps.
```

Example hook: "Most first-time founders spend 6 months building before talking to a single customer. By then it's too late to pivot without burning everything down."

---

### Framework 6 — Case study → Extract → Apply
**Best for:** Analysis threads, company/person breakdowns, trend explainers. High authority signal.

```
Hook:   "How [Company/Person] did [impressive result]:"
Body:   The real story with data and specifics.
Extract: "The pattern underneath this:"
Apply:  "Here's how to use this in your own [context]:"
```

Example hook: "How Notion went from 0 to $10B without ever running a paid ad. Full breakdown:"

## Hook formulas — copy and fill

```
1. "[Result nobody expects] in [timeframe]. Most people think [wrong thing]. Thread 🧵"

2. "I analyzed [large N] of [things]. The pattern every successful one follows:"

3. "Everyone says [common advice]. They're wrong. Here's why:"

4. "[Timeframe] ago: [bad state]. Today: [good state]. The only thing that changed:"

5. "[Big outcome] and I barely [expected action]. Here's what actually drove it:"

6. "Most [audience] get [topic] wrong. Here are [N] mistakes — and how to fix each:"

7. "[Counterintuitive statement]. I learned this the hard way."

8. "A year ago I [failure]. This week I [success]. Thread on what changed 🧵"

9. "[Shocking claim]. Here's the data:"

10. "I [did uncomfortable thing] for [timeframe]. Here's what nobody tells you:"
```

## The hook stress test

Before finalizing a hook, run this check:

1. Would a complete stranger stop scrolling for this with zero context?
2. Does it create a curiosity gap or does it just state the topic?
3. Does it have a specific number, result, or claim — or is it vague?
4. Is it under 220 characters?
5. Does it work as a standalone tweet even if nobody reads the thread?

If the answer to any of these is no — rewrite the hook, not the thread.

## The bookmark test (before posting)

Ask these three questions about the full thread:

1. "Would someone bookmark this to reread later?" → drives bookmark signal (10×)
2. "Would someone reply to argue or agree?" → drives reply signal (13.5×)
3. "Would someone repost this to make themselves look smart?" → drives RT signal (20×)

If none: rewrite the framing. The content is rarely the problem. The framing is.

## Content pillars — rotate across these 4

**Primary expertise** — what you know cold. Teach it concretely, not abstractly.
**Process** — your workflow, what you're building, real-time learnings. Raw > polished.
**Opinion layer** — contrarian takes, what's overrated, what's underrated in your niche.
**Personal texture** — a story, a mistake, a moment. The human stuff that makes people follow a person, not a topic.

Rotate: don't post 3 opinion threads in a row. Mix process + opinion + story.

## Weekly workflow (5–10 hrs/week)

**Monday (30 min):** Idea sourcing. One thing you learned, one mistake, one opinion formed this week = 3 ideas. Pick one. Log rest in a thread bank.

**Tuesday (20 min):** Write the payoff tweet first. If you can't write it, the idea isn't ready.

**Wednesday (15 min):** Hook stress test. Write 3 hooks for the same thread. Pick the sharpest.

**Thursday (45 min):** Post at peak time. For India tech audience: 8–11 AM IST. After posting — reply to every comment for 30 min. Put your link in a reply to yourself.

**Daily (30 min):** Engagement farming. Reply to 5–10 tweets from accounts in your niche. Not "great post" — add a specific data point, a story, or a counterpoint. You're borrowing their audience's attention.

**Sunday (20 min):** Analytics review. Which got the most bookmarks? Replies? Profile clicks? Double down on what won. Kill what consistently underperforms.

## Algorithm penalty checklist

Before posting, confirm none of these are present:

- [ ] Link in the main tweet body → move to a reply
- [ ] 3 or more hashtags → cut to 1–2 or remove
- [ ] "Like if you agree" or other explicit engagement bait → rewrite as a genuine question
- [ ] Cross-platform promotion (pushing to Instagram/YouTube) in the main post → remove or soften
- [ ] Misleading or inflammatory framing that could trigger mass reports → rewrite

## Single tweet rules (non-thread posts)

- One idea only. One sharp angle.
- State the insight, then the why. Or the problem, then the fix. Or the claim, then the contrast.
- End with a question or a provocation — not a summary.
- 150–240 characters is the sweet spot for single tweets (more dwell time than very short, still readable).
- No links in the body for free accounts.
