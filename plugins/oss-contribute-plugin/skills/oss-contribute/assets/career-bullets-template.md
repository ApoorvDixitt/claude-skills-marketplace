# Career Bullet Templates

## Resume Bullet Formula

[Action verb] + [specific thing] + for/to/in [Project] ([scale]) + [result/reference]

### Examples
- Implemented lazy-loading optimization for Next.js (100K+ stars), reducing initial bundle size by 34% (PR #8901, merged)
- Fixed critical race condition in Kubernetes scheduler (CNCF graduated), preventing pod scheduling failures in 500+ node clusters
- Authored authentication documentation for FastAPI (65K+ stars), referenced in 12 Stack Overflow answers
- Added TypeScript strict-mode support to Prisma ORM (35K+ stars), enabling type-safe queries for 200K+ users
- Maintained CI pipeline for React Testing Library (18K+ stars), reducing build time from 12min to 4min

```
Just got a PR merged into [Project]!

The problem: [1-2 sentences]

What I learned: [1-2 sentences]

[Link to PR]

#opensource #[language] #[project]
```

## X/Twitter Thread Formula

```
Tweet 1: "Just shipped my first contribution to [Project] — here's how
a [duration] investigation turned into a [X]-line fix"

Tweet 2: "The issue: [concise]. This affected [who/what]."

Tweet 3: "My approach: [what]. The tricky part was [challenge]."

Tweet 4: "Biggest takeaway: [insight about codebase/process/review]"

Tweet 5: "PR: [link]. If you're looking to contribute, [tip]."
```

## Cold Outreach Formula

```
Subject: [Project] contributor — [observation about their stack]

Hi [Name],

I noticed [Company] uses [Project] in [context]. I've been contributing
for [duration] — most recently [contribution + link].

[One technical insight relevant to their use.]

Would love to chat about [mutual interest]. [Low-commitment ask.]
```

## Notes

- Hand off LinkedIn and X drafts to x-writing and humanizer skills for voice
- Always link the actual PR — verifiable proof beats claims
- Quantify when possible (lines, performance %, users affected)
- Mention the review process to show collaboration ability
